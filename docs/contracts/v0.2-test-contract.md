# v0.2 process-behaviour test contract

This is the v0.2 counterpart to the v0.1 ten-proof test contract. v0.1 proved
the runtime's failure semantics with a fixed set of Erlang tools and a hardcoded
registry seed. v0.2 adds tool manifests, a descriptor registry, and a one-shot
`cli`/port adapter ŌĆö and this doc states the v0.2 process-behaviour proof set in
one place and names the suite and case that proves each line.

Every CLI proof runs through the real `soma_agent_session` ŌåÆ `soma_run` ŌåÆ
`soma_tool_call` process layers, the same path the Erlang tools use. The
contract extends v0.1 without weakening it: a `cli` tool is just another tool
behind the same process boundary.

## How to read this map

Each proof below names:

- **Proof** ŌĆö the process-behaviour guarantee being asserted.
- **Suite** ŌĆö the test suite (EUnit module or Common Test suite) it lives in.
- **Case** ŌĆö the test function that proves it.
- **Entry** ŌĆö the call entry the test enters through, so you can confirm no
  layer is bypassed.

Follow a row to its suite and case and you can read the assertion that proves
the line. That is what makes the contract verifiable by following the map.

## The proof set

### Proof 1 ŌĆö a manifest missing a required field is rejected, and that tool name does not resolve through the registry

The proof has two halves: a missing-field manifest is rejected, *and* that tool
name does not resolve through the running registry.

| Half | Suite | Case | Entry |
| --- | --- | --- | --- |
| Reject | `apps/soma_tools/test/soma_tool_manifest_tests.erl` | `test_normalize_rejects_missing_shared_field` | `soma_tool_manifest:normalize/1` (pure function) |
| Does not resolve | `apps/soma_tools/test/soma_tool_registry_tests.erl` | `test_register_tool_rejects_missing_field_name_unresolvable` | `soma_tool_registry:register_tool/1` on the running registry, then `soma_tool_registry:resolve_descriptor/1` returns `{error, not_found}` |

The does-not-resolve half is the one gap this issue closes with a new EUnit
case: it registers a missing-field manifest, asserts `register_tool/1` returns
the rejection, and asserts `resolve_descriptor/1` for that name returns
`{error, not_found}` ŌĆö proving the registry is left unchanged.

together cover the trail. The contract states this split plainly rather than
papering over it.

### Proof 10 ŌĆö a declared `cli` argv placeholder is preserved on normalize, an undeclared one is rejected

`cli` descriptors may carry whole-argument placeholders (`"{name}"`); each must
name a declared `params` entry. `soma_tool_manifest:normalize/1` preserves the
literal placeholder in the normalized descriptor and rejects the first
placeholder with no matching param as `{error, {unknown_argv_placeholder, Name}}`.

| Half | Suite | Case | Entry |
| --- | --- | --- | --- |
| Declared placeholder preserved | `apps/soma_tools/test/soma_tool_manifest_tests.erl` | `test_normalize_preserves_cli_argv_placeholder_with_declared_param` | `soma_tool_manifest:normalize/1` (pure function) |
| Undeclared placeholder rejected | `apps/soma_tools/test/soma_tool_manifest_tests.erl` | `test_normalize_rejects_cli_argv_placeholder_without_param` | `soma_tool_manifest:normalize/1` (pure function) |

### Proof 11 ŌĆö `cli` argv placeholders render from resolved step args and reach the external process as literal argv

At runtime `soma_run` renders a complete argv from the resolved step input
before starting the worker: it fills each `"{name}"` slot with the resolved
value rendered by its declared type, sends no trailing compatibility input arg,
and fails before any `tool.started` event with `{missing_cli_placeholder, Name}`
when the key is absent ŌĆö the session (and, on the actor path, the actor) stays
alive. Every proof runs through the real `soma_agent_session` ŌåÆ `soma_run` ŌåÆ
`soma_tool_call` layers.

| Proof | Suite | Case | Entry |
| --- | --- | --- | --- |
| Placeholder filled from a prior step's output | `apps/soma_runtime/test/soma_cli_placeholder_SUITE.erl` | `test_cli_argv_placeholder_from_step_replaces_doc` | `soma_agent_session:start_run/2` |
| No trailing compatibility input arg is appended | `apps/soma_runtime/test/soma_cli_placeholder_SUITE.erl` | `test_cli_argv_placeholder_sends_no_trailing_input` | `soma_agent_session:start_run/2` |
| Metacharacter value lands in one literal argv element (no shell) | `apps/soma_runtime/test/soma_cli_placeholder_SUITE.erl` | `test_cli_argv_placeholder_metacharacters_are_one_arg` | `soma_agent_session:start_run/2` ŌåÆ `open_port/2` |
| String / integer / boolean render by declared param type | `apps/soma_runtime/test/soma_cli_placeholder_SUITE.erl` | `test_cli_argv_placeholder_renders_string_integer_boolean` | `soma_agent_session:start_run/2` |
| Missing key fails before `tool.started` with `{missing_cli_placeholder, Name}` | `apps/soma_runtime/test/soma_cli_placeholder_SUITE.erl` | `test_cli_argv_placeholder_missing_key_fails_before_tool_started` | `soma_agent_session:start_run/2` |

## Coverage and the build gate

Every proof above resolves to a real passing case. The one gap ŌĆö the
does-not-resolve half of proof 1 ŌĆö is closed by the new
`test_register_tool_rejects_missing_field_name_unresolvable` case in
`soma_tool_registry_tests`. No other proof is unmapped.

The merge gate runs `rebar3 eunit && rebar3 ct`. Common Test covers the cli
runtime/process guarantees (the cli suites); EUnit covers manifest validation
(`soma_tool_manifest_tests`) and registry behaviour (`soma_tool_registry_tests`).

## Drift caveat

This is a map, not new behaviour, so it can drift: rename a case or delete an
assertion and the doc points at something that no longer proves what it claims.
Every name here is copied from a case that runs in the gate, so a deleted case
shows up as a gate failure, even if not as a doc-drift failure.
