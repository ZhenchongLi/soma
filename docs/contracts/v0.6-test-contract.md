# v0.6 Test Contract — persistent event store via disk_log (opt-in path; survives restart)

This document maps each persistence proof of the v0.6.2 `soma_event_store`
durability slice to the suite and case that proves it. It is the companion to
[v0.2-test-contract.md](v0.2-test-contract.md),
[v0.3-test-contract.md](v0.3-test-contract.md),
[v0.4-test-contract.md](v0.4-test-contract.md), and
[v0.5-test-contract.md](v0.5-test-contract.md).

## What this slice builds

An opt-in persistent path behind the existing `soma_event_store` API. The
in-memory mode is unchanged: `start_link/0` keeps no file and every `by_*` query
keeps reading the in-memory index. A new `start_link/1` taking `#{log => Path}`
opens a `halt`-type `disk_log` at `Path`; `append/2` writes the normalized event
to the log and to the index, and a persistent `init/1` replays the log to
rebuild the index — so a store at a path survives a restart. A truncated or
garbage tail (an unclean shutdown's half-written term) is treated as end-of-log:
boot keeps the intact prefix and finishes `init/1` cleanly. Normalize stays the
single shaping point and the query API does not change.

## Locked design decisions the proofs lean on

1. **Two start functions, one new.** `start_link/0` stays exactly as it is —
   in-memory, no file, no opts; `soma_sup` and every existing caller keep
   working unchanged. `start_link/1` with `#{log => Path}` is the persistent
   opt-in.
2. **The index is a rebuildable cache; the log is the source of truth.** Both
   modes read the in-memory index, so queries never fork. The only behavioural
   fork is in `init/1` and `append/2`.
3. **The on-disk term equals the normalized event.** `append/2` normalizes once,
   then writes that same map to the log and prepends it to the index, so what a
   read-back from disk returns equals what a query returns.
4. **A corrupt tail is skipped, not a crash.** The replay loop keys off how
   `disk_log:chunk/2` reports a bad tail and treats it as end-of-log, keeping
   every intact term read so far.

## Proving suite

One EUnit suite proves this slice's persistence behaviour:

- **`soma_event_store_persist_tests`** — an EUnit module in
  `apps/soma_event_store/test/`, beside the existing `soma_event_store_tests`. A
  persistent store is a plain `gen_server` started with a per-case temp path, so
  it needs no supervision tree; the module drives the store directly through the
  public API and inspects the on-disk log where a proof is about what physically
  sits on disk.

## Persistence proofs → cases

Each criterion maps to its case in `soma_event_store_persist_tests`.

| Criterion | Proof | Case |
| --- | --- | --- |
| 1 | `start_link/0` writes no file over its lifetime and every `by_*` query returns today's results | `test_in_memory_store_writes_no_file_and_queries_unchanged` |
| 2 | `start_link/1` with a path creates a `disk_log` file at `Path` after the first `append/2` | `test_persistent_store_creates_file_after_first_append` |
| 3 | An appended event reads back from the log equal to the store's normalized form of that event | `test_appended_event_reads_back_from_log_as_normalized` |
| 4 | A restart at the same `Path` replays the log so `all/1` returns the events in append order | `test_restart_recovers_events_into_all` |
| 5 | `by_run/2` after a restart returns exactly that run's events in append order, excluding others | `test_by_run_after_restart_filters_to_one_run` |
| 6 | `by_correlation/2` after a restart returns the full cross-layer chain, excluding other correlations | `test_by_correlation_after_restart_returns_full_chain` |
| 7 | A truncated or garbage tail boots without crashing and serves the intact events | `test_truncated_tail_boots_and_serves_intact_events` |

## Out of scope (recorded for the auditor)

- **The v0.6.1 trace proofs are not folded in.** They keep their own home in
  `apps/soma_event_store/test/soma_trace_tests.erl`. Per the issue's open
  question, recording only this slice's persistence proofs satisfies the
  criterion; folding the trace proofs here would churn outside this slice's
  scope.
- **No rotation, compaction, or bounded index.** The halt log grows unbounded
  and the in-memory index still holds every event; bounding memory is a later
  node. This slice buys durability, not a smaller heap.
