## What this slice builds

A disposable worker `soma_llm_call` (in `apps/soma_runtime/src/`), shaped like
`soma_tool_call`: it runs in its own process, performs one call, sends a result
message back to its owner, then exits. The actor owns it directly â€” there is no
`soma_llm_call_sup`, mirroring `soma_run â†’ soma_tool_call` (the decision locked
in [v0.4-test-contract.md](v0.4-test-contract.md)).

## Locked design decisions the proofs lean on

Settled in [../../design-77.md](../../design-77.md):

1. **Mutual exclusion** â€” an envelope carries `steps` (list) **or** `llm` (map),
   never both. Both present is malformed and rejected up front by
   `validate_envelope/1` with `{error, _}`, before any child starts.
2. **Owner-enforced timeout** â€” the actor arms a call-timeout timer when it
   starts the call and treats the timer firing as the timeout; the `slow` mock
   that ignores the timer is exactly the case this proves. Mirrors how
   `soma_run` arms the per-step `state_timeout`.
3. **Actor-driven teardown** â€” on timeout and cancel the actor kills the worker
   itself (`exit(WorkerPid, kill)`), because the bare worker is not a
   `gen_statem` that can drive its own teardown. The mock holds no external OS
   process, so there is nothing to reap beyond the BEAM pid.
4. **Events emitted from the actor** â€” `llm.started`, `llm.succeeded`,
   `llm.failed`, `llm.timeout`, `llm.cancelled` are emitted by the actor (which
   already holds the event-store handle), keeping the worker event-store-free.
   Each carries the task's `correlation_id`, so `by_correlation/2` returns them
   alongside the task's `actor.*` events.
5. **Provider seam** â€” the actual call lives in a single function
   `soma_llm_call:perform_call/1`, the one point a real provider grows into. The
   mock reads a `directive` (`success` / `slow` / `crash` / `hang`) out of the
   `llm` map; there is no provider behaviour or `mock` module yet.

## Proving suites

Two suites prove this slice:

- **`soma_llm_call_tests`** â€” an EUnit module in `apps/soma_runtime/test/`. The
  worker-only proof that touches no actor.
- **`soma_llm_call_SUITE`** â€” a Common Test suite in `apps/soma_actor/test/`,
  set up like `soma_actor_SUITE` (boot the `soma_runtime` app so `soma_run_sup`
  and the event store are alive, start an actor through
  `soma_actor_sup:start_actor/1`, drive it through the real `soma_actor:send/2`
  with an `llm` envelope). Every actor-facing proof.
