## Real issues

- Forwarded calls still grow permanent ingress state. `status_task/3` adds a caller monitor at `apps/soma_actor/src/soma_delegate.erl:79-93`, but it has no request timer. A caller that catches its `gen_server:call` timeout stays alive, so no `DOWN` arrives and `status_requests` keeps the dead request forever. I suspended one coordinator, caught a 50 ms status timeout in the same live process, and the ingress retained one request. Cancellation is worse: `cancel_task/3` stores every caller at lines 104-116 without a monitor or timer. Twenty-five timed-out callers left 25 `cancel_waiters` behind. A slow coordinator turns polling and cancellation into an unbounded leak in the permanent ingress.
- Forced action cancellation acknowledges cleanup before the action tree is dead. The forced-stop handler kills only the round worker at `apps/soma_actor/src/soma_delegate_coordinator.erl:155-173`; its matching `DOWN` enters terminal cleanup at lines 381-431 without waiting for the linked run or tool worker. I suspended a live round worker and its `soma_run`, then cancelled the task. The public call returned `status => cancelled` while the run and `soma_tool_call` pids were both still alive. A stuck run can keep its tool or external process running after the task is publicly terminal and after task leases are released.
- The snapshot boundary still accepts forbidden task state. `validate_new_task/1` at `apps/soma_actor/src/soma_delegate.erl:327-335` checks only total bytes and correlation id, while `round_snapshot/1` at `apps/soma_actor/src/soma_delegate_coordinator.erl:677-688` copies objective, contract, checkpoint, and budgets without validating their terms. The suite itself submits pids, refs, a port, a function, authentication data, and product conversation data under those fields at `apps/soma_actor/test/soma_delegate_SUITE.erl:1540-1550`; all of them enter the worker snapshot. The new result filter also compares only exact lowercase atom or binary key names at `soma_delegate_coordinator.erl:629-653`. I sent a matching result with string key `"authentication_state"` and binary keys `<<"Product_Conversation_Data">>` and `<<"RawLease">>`; the coordinator committed all three and copied them into round 2. This is a spelling blacklist, not a task-only projection.
