## Real issues

- `apps/soma_actor/src/soma_service.erl:569` builds durable task and run ids from `unique_integer/1`. That counter restarts at 1 on every BEAM boot. Two clean boots against one `disk_log` produced `service-task-1` and `service-run-2` twice. The second request overwrote the first task entry and merged both run trails; the old duplicate then returned the new request's result. Durable dedupe is corrupt after the first fresh request following a VM restart. Use restart-stable ids and add a two-VM replay test.
- `apps/soma_actor/src/soma_service.erl:384` restores `max_output_bytes` but drops the persisted `deadline_at_ms`, and `monitor_recovered_run/4` never calls `arm_deadline/1`. Killing the service 100 ms into a 500 ms deadline on a 3000 ms sleep left the adopted task `running` after the deadline. `reconstructed_terminal/2` also maps every recovered `run.cancelled` trail to `cancelled`, ignoring a recorded deadline-expired marker. Owner restart must restore the absolute deadline, re-arm the remaining interval, and preserve the deadline-selected terminal mapping.
- `apps/soma_actor/src/soma_service.erl:500` turns resume-start failure, `nothing_to_do`, and every recovery error into `status => running` without a run pid, monitor, or retry. An accepted-only durable trail reproduced `running` with zero run children and zero terminal events. The normal `start_run` error branch at line 174 leaves the admitted task stuck in the same class. These requests can never reach a terminal state and forever occupy their request ids. Land accepted-before-start and unrecoverable resume failures as bounded terminal data; finish an all-steps-committed trail from its outputs.
- `docs/contracts/RS.1b-test-contract.md:49` claims that `run.started` preserves service budgets. `start_allowed_invocation/5` does not pass `max_output_bytes` or `deadline_at_ms`, `durable_run_options/1` cannot serialize them, and `test_run_started_journals_request_id_and_envelope_hash` checks only request id and hash. The authoritative contract states a guarantee that production and its proof do not provide. Implement and test the stated journal fields, or remove the unsupported guarantee and align the approved design.
